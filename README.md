# fate_app

## Flutter SDK (FVM)

Версия Flutter зафиксирована в [`.fvm/fvm_config.json`](.fvm/fvm_config.json). Нужен [FVM](https://fvm.app/):

```bash
fvm install
fvm flutter pub get
```

Дальше используйте `fvm flutter` / `fvm dart` вместо глобальных `flutter` / `dart`. В VS Code / Cursor путь к SDK уже задан в [`.vscode/settings.json`](.vscode/settings.json).

## Описание проекта

**fate_app** - это приложение на Flutter, разработанное для управления персонажами в ролевых играх по системе FAE. Оно предоставляет пользователям возможность создавать, редактировать и просматривать персонажей, а также управлять их характеристиками, навыками и другими аспектами.

## Основные функции

- **Создание персонажей**: Пользователи могут добавлять новых персонажей с уникальными характеристиками.
- **Редактирование персонажей**: Возможность изменять информацию о персонаже, включая имя, концепцию и навыки.
- **Генерация PDF**: Создание PDF-документов с информацией о персонаже для удобного хранения и печати.
- **ИИ (черновик и поля)**: сначала **Groq** (`GROQ_API_KEY`), при ошибке — **OpenRouter** (`OPENROUTER_API_KEY`). Ключи задаются в `.env` или `--dart-define`, отдельного экрана настроек в приложении нет.
- **Аватар**: кнопка «Сгенерировать аватар» — портрет по **имени**, **концепту** и **описанию** листа (нужно хотя бы одно из полей) через [SiliconFlow Images API](https://docs.siliconflow.com/en/api-reference/images/images-generations). Нужен **`SILICONFLOW_API_KEY`** в `.env` или `--dart-define`; модель по умолчанию **`black-forest-labs/FLUX.1-schnell`** (переопределение: `SILICONFLOW_AVATAR_MODEL`).

## Архитектура проекта

Проект написан по **Clean Architecture**. Основные слои:

- **Presentation**: Взаимодействует только через use case.
- **Domain**: Взаимодействует с data через repository.
- **Data**: Содержит реализацию доступа к данным.

### Дополнительное описание струтуры

- **core**: Содержит общие вещи, которые не относятся к указанным слоям или относятся ко всем слоям сразу.
- **app_size.dart**: Отвечает за адаптивные размеры в приложении, включает расширения для `int` и `double`, чтобы упростить задание размеров.
- **router**: Использует `go_router` для управления навигацией.
- **Dependency Injection**: Реализован с помощью `get_it`.
- **Mapping**: Использует `dart_mappable` (для генерации используйте команду: `./rebuild.bat`).
- **Обработка ошибок**: Использует `dartz` на уровне домена.
- **Локальная база данных**: Хранит персонажей с помощью `sqflite`.
- **Экспорт PDF**: Реализован через библиотеку `pdf`.
